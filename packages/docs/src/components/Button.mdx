# Bouton

Le **bouton** est un élément d'interaction avec l'interface permettant à
l'utilisateur d'effectuer une action.

Il existe quatre variantes :

- **primary** - pour définir une action principale
- **secondary** - pour contraster avec l'action principale
- **ghost** - pour des actions plus discrètes mais nécessaires
- **danger** - pour les actions destructives (suppression, retrait…)

Décliné en trois tailles afin de s'adapter aux différents contextes :

- **sm**
- **md** (par défaut)
- **lg**

## Bonnes pratiques

Bien différencier les boutons des liens :

- un **bouton** déclenche une action (sur la page courante, soumission, ouverture de modale)
- un **lien** permet de naviguer vers une autre page

Pour garantir une bonne hiérarchisation des actions :

- limiter l'usage du bouton **primary à un seul par écran** (l'action prioritaire)
- ne jamais placer deux boutons primary côte à côte
- privilégier une combinaison **primary + secondary** ou **primary + ghost**
- utiliser **danger** uniquement pour des actions irréversibles, idéalement précédées d'une confirmation

Le bouton secondary est utilisé lorsque l'action est moins prioritaire, par exemple pour réinitialiser un formulaire ou annuler.

## Composition

Côté React et Angular, le bouton accepte du **contenu projeté** :

- texte simple
- texte + icône Lucide (`<ori-button><lucide-icon [img]="SendIcon" /> Envoyer</ori-button>`)
- icône seule (penser à fournir un `aria-label`)
